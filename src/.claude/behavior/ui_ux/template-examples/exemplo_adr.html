Contexto

O projeto RAG Corporativo precisa armazenar documentos como grafo de conhecimento E realizar busca vetorial por similaridade semantica. A questao era: usar uma base para o grafo (Neo4j) + outra para vetores (Pinecone/Weaviate), ou unificar tudo no Neo4j com seus vector indexes nativos?

Decisao Detalhada
Aprovada

Neo4j unifica grafo + vetores. Vantagem principal: query unica com Cypher combina travessia de grafo + busca vetorial. Exemplo de query hibrida:

{
  "query": "CALL db.index.vector.queryNodes('chunk_embeddings', 10, $queryVector)
    YIELD node, score
    WHERE node.confidentiality IN ['public','internal']
    MATCH (node)-[:PART_OF]->(doc:Document)-[:BELONGS_TO]->(sys:System)
    RETURN doc.title, node.text, sys.name, score
    ORDER BY score DESC",
  "parameters": { "queryVector": "[0.12, -0.45, 0.78, ...]" }
}

Consequencias
Positivas
  • Stack unificado (1 base)
  • Query hibrida nativa (grafo+vetor)
  • Menor custo operacional
Negativas
  • Performance vetorial ~15% inferior a bases especializadas
  • Vendor lock-in no Neo4j
Se o volume de chunks ultrapassar 5M, avaliar performance do vector index vs base especializada
Neo4j Aura (cloud) tem limitacoes de vector index em tiers menores
